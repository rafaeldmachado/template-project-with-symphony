into actionable requirements that agents can implement.

## Template

When creating a new product spec:

```markdown
# <Feature Name>

## Problem
What user problem does this solve?

## Solution
What are we building? Describe the user experience.

## Requirements
- Functional requirements (what it must do)
- Non-functional requirements (performance, accessibility, security)

## Acceptance criteria
Specific, testable conditions. An agent will use these as a checklist.

## Out of scope
What this feature intentionally does NOT include.

## Design references
Links to mockups, wireframes, or design system components.
```
